# Welcome to OpenAgreements

OpenAgreements is an open-source CLI tool and TypeScript library for filling standard legal agreement templates and producing professionally formatted, signable DOCX files. Built by the team behind [UseJunior.com](https://usejunior.com) and used in production at Am Law 100 firms.

## How It Works

<Steps>
  <Step title="Choose a template">
    Select from 28+ standard agreements covering NDAs, cloud terms, SAFEs, NVCA financing documents, and more.
  </Step>
  <Step title="Fill in your details">
    Answer interactive prompts or provide data via JSON file. AI agents like Claude Code can interview you automatically.
  </Step>
  <Step title="Get a professionally formatted DOCX">
    OpenAgreements preserves all original formatting while filling in your values, producing a signable document ready for review.
  </Step>
</Steps>

## Template Sources

OpenAgreements includes templates from trusted sources:

- **[Common Paper](https://commonpaper.com)** — Cloud service agreements, NDAs, data processing agreements (CC BY 4.0)
- **[Bonterms](https://bonterms.com)** — Professional services, mutual NDAs (CC BY 4.0)
- **[Y Combinator](https://www.ycombinator.com/documents)** — SAFE agreements (CC BY-ND 4.0)
- **[NVCA](https://nvca.org/model-legal-documents/)** — Financing documents downloaded at runtime

<Note>
OpenAgreements does not provide legal advice. No affiliation with or endorsement by template sources is implied. Consult an attorney for legal guidance.
</Note>

## Execution Modes

OpenAgreements supports two execution modes:

<Tabs>
  <Tab title="Hosted Remote MCP">
    Fast setup via hosted MCP connector at `https://openagreements.ai/api/mcp`. Best for non-sensitive documents and quick workflows.
  </Tab>
  <Tab title="Fully Local">
    Run locally via `npx`, global install, or local stdio MCP package. Best for sensitive documents and air-gapped environments.
  </Tab>
</Tabs>

<Info>
Choose based on document sensitivity, internal policy, and workflow needs. See the [Trust Checklist](/advanced/trust-checklist) for a 60-second data flow summary.
</Info>
