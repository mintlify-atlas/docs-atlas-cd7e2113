## Overview

The OpenAgreements checklist system uses RFC 6902 JSON Patch operations for safe, auditable updates with revision conflict detection. The patch system provides:

- **Two-phase apply**: Validate first, then apply with proof of validation
- **Revision tracking**: Prevent concurrent update conflicts
- **Dry-run simulation**: Validate patches produce valid schemas
- **Idempotent replay**: Safe re-application of already-applied patches
- **Proposed mode**: Stage patches without applying them

## Patch Operations

```typescript
interface ChecklistPatchOperation {
  op: 'add' | 'replace' | 'remove';
  path: string;  // JSON Pointer (RFC 6901)
  value?: unknown;  // Required for add/replace
  rationale?: string;
  source?: string;
}
```

<ParamField path="op" type="enum" required>
  - `add`: Add a new value (can append to arrays with `/-`)
  - `replace`: Replace an existing value
  - `remove`: Remove an existing value
</ParamField>

<ParamField path="path" type="string" required>
  JSON Pointer to the target (e.g., `/checklist_entries/E-1/status`)
</ParamField>

<ParamField path="value" type="unknown">
  Value for add/replace operations
</ParamField>

<ParamField path="rationale" type="string">
  Human-readable explanation of why this change is being made
</ParamField>

<ParamField path="source" type="string">
  Source identifier (e.g., "DOCX import", "user edit")
</ParamField>

### JSON Pointer Format

Paths use RFC 6901 JSON Pointer syntax:
- Start with `/`
- Segments separated by `/`
- Escape `~` as `~0` and `/` as `~1`
- Array indices are integers (0-based)
- Use `/-` to append to arrays

Examples:
```
/checklist_entries/E-1/status
/documents/DOC-1/primary_link
/action_items/A-1/assigned_to/individual_name
/checklist_entries/E-1/citations/-
/checklist_entries/E-1/citations/0
```

### Operation Rules

<Warning>
  - Root path operations (`/`) are not supported
  - Dangerous keys (`__proto__`, `constructor`, `prototype`) are rejected
  - `remove` and `replace` cannot target the append index `/-`
  - Array indices must be in bounds
  - Parent paths must exist and be containers (object or array)
</Warning>

### Citation Operations

When adding/replacing citations, the value must match `PatchCitationSchema`:

```typescript
interface PatchCitation {
  text: string;  // Required
  link?: string;
  filepath?: string;
}
```

Example:
```typescript
{
  op: 'add',
  path: '/checklist_entries/E-1/citations/-',
  value: {
    text: 'Section 4.2',
    link: 'https://docs.example.com/pa.pdf#page=12'
  },
  rationale: 'Added reference to purchase agreement'
}
```

## Validation Error Codes

| Code | Description |
|------|-------------|
| `CHECKLIST_SCHEMA_INVALID` | Input checklist failed schema validation |
| `PATCH_SCHEMA_INVALID` | Patch envelope failed schema validation |
| `REVISION_CONFLICT` | expected_revision doesn't match current_revision |
| `TARGET_NOT_FOUND` | Path references non-existent key or array index |
| `TARGET_PATH_INVALID` | Invalid JSON Pointer or dangerous key |
| `TARGET_PARENT_INVALID` | Parent path is not a container |
| `POST_PATCH_SCHEMA_INVALID` | Resulting checklist would violate schema |

## Apply Error Codes

| Code | Description |
|------|-------------|
| `APPLY_REQUEST_SCHEMA_INVALID` | Request payload invalid |
| `VALIDATION_REQUIRED` | validation_id missing or expired |
| `VALIDATION_MISMATCH` | Patch hash doesn't match validated hash |
| `PATCH_ID_CONFLICT` | patch_id already exists with different hash |
| `APPLY_OPERATION_FAILED` | Operation failed during apply |

## Example Workflow

### Basic Update Flow

```typescript
import {
  validateChecklistPatch,
  applyChecklistPatch,
  type ClosingChecklist,
  type ChecklistPatchEnvelope
} from 'openagreements';

const checklist: ClosingChecklist = { /* ... */ };
const currentRevision = 5;

const patch: ChecklistPatchEnvelope = {
  patch_id: 'patch-001',
  expected_revision: 5,
  mode: 'APPLY',
  operations: [
    {
      op: 'replace',
      path: '/checklist_entries/E-1/status',
      value: 'FULLY_EXECUTED',
      rationale: 'All signatures received'
    }
  ]
};

// Step 1: Validate
const validationResult = await validateChecklistPatch({
  checklist_id: 'checklist-123',
  checklist,
  current_revision: currentRevision,
  patch
});

if (!validationResult.ok) {
  console.error('Validation failed:', validationResult.diagnostics);
  return;
}

console.log('Validation succeeded:', validationResult.validation_id);

// Step 2: Apply
const applyResult = await applyChecklistPatch({
  checklist_id: 'checklist-123',
  checklist,
  current_revision: currentRevision,
  request: {
    validation_id: validationResult.validation_id,
    patch
  }
});

if (!applyResult.ok) {
  console.error('Apply failed:', applyResult.message);
  return;
}

console.log('Applied patch:', applyResult.patch_id);
console.log('New revision:', applyResult.new_revision);
console.log('Updated checklist:', applyResult.checklist);
```

### Proposed Mode (Review Workflow)

```typescript
const proposedPatch: ChecklistPatchEnvelope = {
  patch_id: 'patch-002',
  expected_revision: 5,
  mode: 'PROPOSED',  // Don't apply yet
  operations: [
    {
      op: 'replace',
      path: '/checklist_entries/E-2/responsible_party/organization',
      value: 'New Corp',
      rationale: 'Change of control'
    }
  ]
};

// Validate and stage (doesn't increment revision)
const validationResult = await validateChecklistPatch({
  checklist_id: 'checklist-123',
  checklist,
  current_revision: 5,
  patch: proposedPatch
});

const applyResult = await applyChecklistPatch({
  checklist_id: 'checklist-123',
  checklist,
  current_revision: 5,
  request: {
    validation_id: validationResult.validation_id,
    patch: proposedPatch
  }
});

// applyResult.applied === false
// applyResult.new_revision === 5 (unchanged)
// Patch is recorded in proposed_patch_store for later approval
```

### Handling Revision Conflicts

```typescript
const patch: ChecklistPatchEnvelope = {
  patch_id: 'patch-003',
  expected_revision: 5,  // But current is 6
  mode: 'APPLY',
  operations: [/* ... */]
};

const result = await validateChecklistPatch({
  checklist_id: 'checklist-123',
  checklist,
  current_revision: 6,  // Mismatch!
  patch
});

// result.ok === false
// result.diagnostics[0].code === 'REVISION_CONFLICT'
// User must fetch latest revision and retry
```
