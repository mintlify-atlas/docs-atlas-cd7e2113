## Execution Modes

OpenAgreements supports two trust boundaries:

### Remote MCP (Hosted)

- Zero local dependencies
- Fast setup in Claude Desktop or compatible clients
- Server handles DOCX generation
- Endpoint: `https://openagreements.ai/api/mcp`
- Best for: Quick setup, non-sensitive documents

### Local Execution

- Fully local package execution via `npx` or global install
- No network calls during template filling
- Local stdio MCP servers available
- Best for: Sensitive documents, internal policy compliance

<Note>
  Choose based on document sensitivity and workflow speed needs. See the [trust checklist](https://github.com/open-agreements/open-agreements/blob/main/docs/trust-checklist.md) for data-flow details.
</Note>

## Quick Start by Tool

<Tabs>
  <Tab title="Claude Code">
    Install the agent skill:
    ```bash
    npx skills add open-agreements/open-agreements
    ```
    Then ask Claude to draft an agreement.
  </Tab>
  <Tab title="Gemini">
    Install the CLI extension:
    ```bash
    gemini extensions install https://github.com/open-agreements/open-agreements
    ```
    Then ask Gemini to draft an agreement.
  </Tab>
  <Tab title="MCP">
    Add to your MCP configuration:
    ```json
    {
      "mcpServers": {
        "open-agreements": {
          "url": "https://openagreements.ai/api/mcp"
        }
      }
    }
    ```
  </Tab>
  <Tab title="CLI">
    Install globally:
    ```bash
    npm install -g open-agreements
    ```
    List templates:
    ```bash
    open-agreements list
    ```
  </Tab>
</Tabs>
