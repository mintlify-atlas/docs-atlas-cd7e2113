Get started with OpenAgreements and fill your first legal template in under 2 minutes. No installation required.

## Prerequisites

You'll need Node.js version 20 or higher installed on your machine.

<Info>
OpenAgreements works best with AI assistants like Claude Code or Gemini, but you can also use it directly from the command line.
</Info>

## Quick Start with npx (Zero Install)

The fastest way to use OpenAgreements is with `npx`, which runs the CLI without requiring installation:

<Steps>
  <Step title="List available templates">
    See all 28+ available templates:

    ```bash
    npx -y open-agreements@latest list
    ```

    For machine-readable JSON output (useful for AI agents):

    ```bash
    npx -y open-agreements@latest list --json
    ```
  </Step>

  <Step title="Fill a template">
    Let's fill a Common Paper Mutual NDA. You can provide values inline:

    ```bash
    npx -y open-agreements@latest fill common-paper-mutual-nda \
      --set party_1_name="Acme Corp" \
      --set party_2_name="Beta Inc" \
      --set purpose="Exploring a potential partnership" \
      --set effective_date="2024-03-15" \
      --set governing_law="Delaware" \
      -o my-nda.docx
    ```

    Or use a JSON data file:

    ```bash
    npx -y open-agreements@latest fill common-paper-mutual-nda \
      -d values.json \
      -o my-nda.docx
    ```
  </Step>

  <Step title="Open your DOCX">
    Open the generated `my-nda.docx` file in Microsoft Word, Google Docs, or LibreOffice to review and sign.

    <Check>
    All original formatting, styles, and document structure are preserved!
    </Check>
  </Step>
</Steps>

## Example: Using a JSON Data File

Create a `values.json` file with your template data:

```json values.json
{
  "party_1_name": "Acme Corp",
  "party_2_name": "Beta Inc",
  "purpose": "Exploring a potential partnership for Widget development",
  "effective_date": "2024-03-15",
  "governing_law": "Delaware"
}
```

Then fill the template:

```bash
npx -y open-agreements@latest fill common-paper-mutual-nda \
  -d values.json \
  -o acme-beta-nda.docx
```

## Discovering Template Fields

To see what fields a template requires, use the `list` command with `--json` and filter by template:

```bash
npx -y open-agreements@latest list --json | \
  jq '.[] | select(.name == "common-paper-mutual-nda") | .fields'
```

This shows all fields with their types, descriptions, and whether they're required:

```json
[
  {
    "name": "party_1_name",
    "type": "text",
    "description": "Name of the first party",
    "required": true
  },
  {
    "name": "party_2_name",
    "type": "text",
    "description": "Name of the second party",
    "required": true
  },
  {
    "name": "effective_date",
    "type": "date",
    "description": "Effective date of the agreement",
    "required": true
  }
]
```

## Using with AI Assistants

OpenAgreements works seamlessly with AI assistants. Just ask Claude Code or Gemini to draft an agreement:

<Tabs>
  <Tab title="Claude Code">
    ```bash
    # Install the agent skill
    npx skills add open-agreements/open-agreements
    ```

    Then ask Claude:
    ```
    Draft an NDA between Acme Corp and Beta Inc
    ```
  </Tab>

  <Tab title="Gemini CLI">
    ```bash
    # Install the extension
    gemini extensions install https://github.com/open-agreements/open-agreements
    ```

    Then ask Gemini to draft an agreement.
  </Tab>

  <Tab title="Direct Usage">
    AI assistants can run OpenAgreements directly with npx - no installation needed:

    ```
    Use npx open-agreements to help me fill a mutual NDA
    ```
  </Tab>
</Tabs>
