OpenAgreements can be used without installation via `npx`, or installed globally for faster repeated use.

## Requirements

<Card title="Node.js >= 20" icon="node">
  OpenAgreements requires **Node.js version 20 or higher**.
</Card>

Check your Node.js version:

```bash
node --version
```

If you need to install or upgrade Node.js, visit [nodejs.org](https://nodejs.org/).

## Installation Methods

<Tabs>
  <Tab title="npx (Recommended)">
    ### Use Without Installation

    Run OpenAgreements directly without installing it globally:

    ```bash
    npx open-agreements list
    npx open-agreements fill common-paper-mutual-nda -d values.json -o output.docx
    ```

    **Pros:**
    - No installation required
    - Always uses the latest version
    - Zero configuration
    - Works immediately in CI/CD pipelines

    **Cons:**
    - Slightly slower first run (downloads package on-demand)
    - Requires `-y` flag to skip confirmation in scripts

    <Tip>
    For scripts and automation, use `npx -y open-agreements` to skip the confirmation prompt.
    </Tip>
  </Tab>

  <Tab title="Global Installation">
    ### Install Globally

    Install OpenAgreements globally for faster repeated use:

    <CodeGroup>

    ```bash npm
    npm install -g open-agreements
    ```

    ```bash yarn
    yarn global add open-agreements
    ```

    ```bash pnpm
    pnpm add -g open-agreements
    ```

    </CodeGroup>

    After installation, the `open-agreements` command is available system-wide:

    ```bash
    open-agreements list
    open-agreements fill common-paper-mutual-nda -d values.json -o output.docx
    ```

    **Pros:**
    - Faster execution (no download on each run)
    - Shorter command (no `npx` prefix)
    - Better for frequent use

    **Cons:**
    - Requires manual updates to get new versions
    - May conflict with different project-specific versions
  </Tab>

  <Tab title="Project Dependency">
    ### Add to Your Project

    Install OpenAgreements as a project dependency to use it as a library or CLI tool in your codebase:

    <CodeGroup>

    ```bash npm
    npm install open-agreements
    ```

    ```bash yarn
    yarn add open-agreements
    ```

    ```bash pnpm
    pnpm add open-agreements
    ```

    </CodeGroup>

    Use via npm scripts in `package.json`:

    ```json package.json
    {
      "scripts": {
        "fill-template": "open-agreements fill common-paper-mutual-nda -d data.json -o output.docx"
      }
    }
    ```

    Or import in TypeScript/JavaScript:

    ```typescript
    import { fillTemplate, loadMetadata } from 'open-agreements';
    ```

    See the [API Reference](/api/overview) for programmatic usage.
  </Tab>
</Tabs>

## Verify Installation

Check that OpenAgreements is working correctly:

<CodeGroup>

```bash npx
npx open-agreements --version
```

```bash global
open-agreements --version
```

</CodeGroup>

Expected output:
```
0.4.0
```

List available templates to confirm everything is working:

```bash
npx open-agreements list
```

You should see a table of 28+ templates.

## Updating OpenAgreements

<Tabs>
  <Tab title="npx">
    ### Always Latest with npx

    When using `npx`, you automatically get the latest version. No manual updates needed.

    To force a specific version:

    ```bash
    npx open-agreements@0.4.0 list
    ```

    To use the latest version explicitly:

    ```bash
    npx open-agreements@latest list
    ```
  </Tab>

  <Tab title="Global Installation">
    ### Update Global Installation

    <CodeGroup>

    ```bash npm
    npm update -g open-agreements
    ```

    ```bash yarn
    yarn global upgrade open-agreements
    ```

    ```bash pnpm
    pnpm update -g open-agreements
    ```

    </CodeGroup>

    Check the installed version:

    ```bash
    open-agreements --version
    ```

  </Tab>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Node.js version too old">
    If you see an error about Node.js version:

    ```
    error: engine "node" is incompatible with this module
    ```

    **Solution:** Upgrade to Node.js 20 or higher. Download from [nodejs.org](https://nodejs.org/).
  </Accordion>

  <Accordion title="npx hangs or times out">
    If `npx` appears to hang:

    **Solution:** Check your internet connection, or try installing globally instead.

    If behind a corporate proxy, configure npm:
    
    ```bash
    npm config set proxy http://proxy.company.com:8080
    npm config set https-proxy http://proxy.company.com:8080
    ```
  </Accordion>

  <Accordion title="Permission denied on global install">
    If you see permission errors during global installation:

    ```
    EACCES: permission denied
    ```

    **Solution:** Use a Node version manager like [nvm](https://github.com/nvm-sh/nvm) or [fnm](https://github.com/Schniz/fnm), or configure npm to use a different global directory:

    ```bash
    mkdir ~/.npm-global
    npm config set prefix '~/.npm-global'
    echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
    source ~/.bashrc
    ```

    Then try installing again.
  </Accordion>

  <Accordion title="Command not found after global install">
    If `open-agreements` command is not found after global installation:

    **Solution:** Ensure npm's global bin directory is in your PATH:

    ```bash
    npm config get prefix
    ```

    Add the bin directory to your PATH in `~/.bashrc` or `~/.zshrc`:

    ```bash
    export PATH="$(npm config get prefix)/bin:$PATH"
    ```

    Reload your shell:

    ```bash
    source ~/.bashrc  # or ~/.zshrc
    ```
  </Accordion>
</AccordionGroup>
